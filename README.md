1) top 명령
top 명령어는 현재 OS의 상태를 나타내주는 CLI 어플리케이션입니다. 메모리 사용량,CPU 사용량 등을 나타내주면 top을 실행하는 동안에는 주기적인 업데이트로 실시간에 근접한 내용을 보여줍니다.top 명령어를 입력하면 이미지와 같이 출력된다. 
top 명령어를 사용했을 때 나오는 결과는 1.시스템 시간,uptime,유저 세션 수 - 가장 왼쪽 위의 시간은 시스템 현재 시간을 나타내고 바로 이어서 OS가 Up된 시간인 uptime이 나온다 그 옆에는 현재 접속 중인 유저 세션 수가 나온다.
2. 로드 애버리지 - 로드 애버리지 영역에는 3개의 숫자가 출력된다. 왼쪽에서부터 각각 1분,5분,15분 동안의 평균 시스템 부하를 나타낸다.
3. 3. Tasks - Tasks는 현재 프로세스들의 상태를 출력한다.
4. CPU사용량 - %Cpu(s)는 CPU의 사용율을 출력한다. 총합이 100%이며 각 항목은 다음과 같다. 
- us : 유저 영역에서의 CPU 사용율
- sy : 커널 영역에서의 CPU 사용율
- ni : 프로세스 우선순위 설정에 사용되는 CPU 사용율
- id : 사용하지 않는 CPU 비율
- wa : I/O 대기 상태의 CPU 사용율
- hi : 하드웨어 인터럽트에 사용되는 CPU 사용율
- si : 소프트웨어 인터럽트에 사용되는 CPU 사용율
- st : Hypervisor VM에서 사용하여 대기하는 CPU 비율
- 5. 메모리 사용량
- | 항목       | 설명                                                         |
|------------|--------------------------------------------------------------|
| MiB Mem    | RAM의 정보를 출력한다.                                        |
| MiB Swap   | 디스크를 메모리처럼 사용하는 Swap 메모리 영역의 정보를 출력한다. |
| total      | 총 메모리 양                                                 |
| free       | 사용 가능한 메모리 양                                        |
| used       | 사용 중인 메모리 양                                         |
| buff/cache | 커널 버퍼와 디스크의 페이지 캐시에 사용되는 메모리 양 (I/O에 사용되는 버퍼 메모리) |                               
2) ps 명령
  ps명령어는 process state의 약자로 현재 실행중인 프로세스와 상태를 출력하는 명령어입니다. man ps를 사용 시 ps 명령어와 관련된 메뉴얼을 볼 수 있는데,ps 명령어의 옵션은 각 시스템 계열 V,BSD,GNU마다 다른 표기법 및 출력을 가지고 있습니다.
  -ps: 사용자와 관련된 프로세스만 출력합니다.
  
  2. ps 명령어 출력 항목
  | 항목 | 시스템 유형 | 설명 |
|------|-------------|-------|
| USER | BSD | 프로세스 소유자의 이름 |
| UID | System V | 프로세스 소유자의 사용자 ID |
| PID | 공통 | 프로세스 ID |
| PPID | 공통 | 부모 프로세스 ID |
| %CPU | BSD | CPU 사용 비율 |
| %MEM | BSD | 메모리 사용 비율 |
| VSZ | 공통 | 가상 메모리 사용량(KB) |
| RSS | 공통 | 실제 메모리 사용량 |
| TTY | 공통 | 연결된 터미널 |
| S | System V | 상태 코드 |
| STAT | BSD | 상태 코드 |
| TIME | 공통 | 누적 CPU 시간 |
| COMMAND | 공통 | 실행 명령 |
| STIME | 공통 | 시작 시간 |
| C | System V | CPU 사용률(단기) |
| CP | BSD | CPU 사용률(단기) |
| F | 공통 | 플래그 |
| PRI | 공통 | 실행 우선순위 |
| NI | 공통 | nice 우선순위 |

3. 명령어 옵션
-A : 모든 프로세스를 출력
a (BSD) : 터미널과 연관된 프로세스를 출력. x 옵션과 함께 전체 프로세스 출력 시 사용
-a : 세션 리더를 제외하고 터미널에 종속되지 않은 모든 프로세스를 출력
-e : 커널 프로세스를 제외한 모든 프로세스를 출력
-f : 풀 포맷으로 출력. UID, PID, PPID 등이 함께 표시
-l (System V), l (BSD) : 긴 포맷으로 출력. PRI, NI 값 확인 가능
-o : 출력 포맷을 지정
-M : 64비트 프로세스를 출력
-m : 프로세스와 커널 스레드를 함께 출력
-p : 특정 PID만 출력
-r : 현재 실행 중인 프로세스 출력
u (BSD) : 사용자 기준으로 프로세스 출력
-u(사용자) : 특정 사용자의 프로세스 출력
x (BSD) : 터미널에 종속되지 않은 프로세스 출력
-x : 로그인 후 종료되지 않은 프로세스 출력 (터미널 없이 실행되는 프로세스 확인 가능)

3) jobs 명령어
   jobs = 기본적으로 jobs 명령어는 현재 세션에서 실행 중이거나 중단된 작업 목록을 출력합니다. 옵션 없이 사용하면 현재 활성화된 작업 목록이 표시됩니다.
1. 명령어
-l: 각 작업의 프로세스 ID(PID)도 함께 출력합니다.
-p: 각 작업의 프로세스 ID만 출력합니다.
-n: 가장 최근에 상태가 변경된 작업만 출력합니다.
-r: 실행 중(Running)인 작업만 출력합니다.
-s: 일시 중지된 작업만 출력합니다.

2. 명령어의 주요기능
: jobs 명령어는 쉘 세션에서 실행 중이거나 일시 중지된 모든 작업을 확인할 수 있는 매우 유용한 도구입니다. 이를 통해 사용자는 각 작업의 작업 번호를 확인하고, 이를 기반으로 fg,bg,kill 명령어를 사용하여 작업을 포그라운드로 전환하거나 백그라운드로 전환, 또는 종료할 수 있습니다. jobs 명령어는 특히 다수의 작업을 동시에 처리할 때 효과적으로 관리할 수 있도록 도와줍니다.

3. 작업 제어 관련 명령어
fg: 백그라운드 또는 일시 중지된 작업을 포그라운드로 전환합니다.
bg: 일시 중지된 작업을 백그라운드에서 다시 실행합니다.
kill: 특정 작업을 종료합니다.

4) kill 명령어
1. kill 명령어
: kill 명령어는 지정한 프로세스에 지정한 시그널을 보냅니다. 시스템에 문제가 생겨 해당 프로세스를 종료할 필요가 있을 경우 많이 사용됩니다. 단순 kill 명령어로 종료가 되지 않는 프로세스는 SIGKILL 시그널을 보내어 강제로 종료시킵니다.
2. kill 옵션
   -15: 정상 종료
   TERM 시그널 (default Option 으로 기본 적용)
   자신이 하던 작업을 모두 안전하게 종료하는 절차를 밟으며 프로세스를 종료
   메모리상에 있는 데이터와 각종 설정/환경 파일을 안전하게 저장한 후 프로세스를 종료

   -9: 강제 종료
   리눅스 커널이 프로세스를 강제 종료
   프로세스를 강제 종료하기 때문에 저장되지 않은 데이터가 날아가는 경우 발생
3. signal 종류
 SIGHUP (1) – 터미널 연결 종료 / 설정 재로드
SIGINT (2) – 인터럽트(Ctrl + C)
SIGQUIT (3) – 종료(Ctrl + ) + 코어덤프 생성
SIGILL (4) – 잘못된 명령어 실행
SIGTRAP (5) – 디버깅 트랩
SIGABRT (6) – abort() 호출로 비정상 종료
SIGBUS (7) – 버스 에러
SIGFPE (8) – 산술 오류(0으로 나누기 등)
SIGKILL (9) – 강제 종료(무조건 종료)
SIGUSR1 (10) – 사용자 정의 신호 1
SIGSEGV (11) – 세그멘테이션 오류(잘못된 메모리 접근)
SIGUSR2 (12) – 사용자 정의 신호 2
SIGPIPE (13) – 파이프에 쓰기 오류
SIGALRM (14) – 타이머 시간 만료
SIGTERM (15) – 정상 종료 요청
SIGSTKFLT (16) – 스택 오류(일부 시스템)
SIGCHLD (17) – 자식 프로세스 종료 감지
SIGCONT (18) – 정지된 프로세스 재개
SIGSTOP (19) – 프로세스 정지(무시 불가)
SIGTSTP (20) – 터미널 정지(Ctrl + Z)
